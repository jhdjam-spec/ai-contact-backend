# ADR-0005: Абстракция email-отправителя (console / smtp)

## Status
Accepted
Date: 2026-07-22 · Deciders: @jhdjam-spec

## Context

ТЗ требует отправку двух писем (владельцу + копия пользователю). Но проверяющий
запускает проект без нашего SMTP; реальная отправка при демо рискует уходить в
спам или требовать app-password. Нужно показать факт отправки двух писем без
обязательной настройки почты.

## Decision Drivers

* Обязательно: два письма (владельцу + копия пользователю).
* Обязательно: факт отправки виден проверяющему без настройки SMTP.
* Обязательно: сбой одного письма не отменяет второе.
* Желательно: реальный SMTP включается одной настройкой.

## Considered Options

### Вариант 1: Только реальный SMTP
- Плюсы: «по-настоящему».
- Минусы: требует app-password/креды; риск спама; проверяющему сложнее.

### Вариант 2: Абстракция + console-бэкенд (дефолт) + SMTP по флагу
- Плюсы: письма видны как файлы `.eml` без SMTP; реальный SMTP — одной env;
  симметрия с AI-абстракцией.
- Минусы: console не проверяет реальную доставку.

### Вариант 3: Сторонний sandbox (Mailtrap) по умолчанию
- Плюсы: письма видны в веб-интерфейсе.
- Минусы: нужен внешний аккаунт/токен у проверяющего.

## Decision

Выбрали **Вариант 2**. Интерфейс `EmailSender.send()`; `ConsoleEmailSender`
(дефолт, пишет `.eml` в `logs/emails/` + лог) и `SMTPEmailSender` (aiosmtplib,
флаг `EMAIL_BACKEND=smtp`). Письма отправляются независимо (`asyncio.gather`),
сбой каждого логируется.

## Rationale

Console-бэкенд даёт проверяемый факт двух писем без внешних зависимостей и
аккаунтов, а реальный SMTP включается одной переменной. Подход симметричен
AI-абстракции (ADR-0003) — единый способ работы с внешними сервисами.

## Consequences

- Positive: факт отправки проверяется без настройки почты; реальный SMTP — одной
  env; независимость писем повышает устойчивость.
- Negative: console-бэкенд не подтверждает реальную доставку (осознанный
  компромисс для демо/тестового).
- Risks + Mitigation: подмена бэкенда в проде → выбор инкапсулирован в
  `_build_sender`, покрыт тестом на создание двух `.eml`.
