## 6. Масштабирование (осознанные ограничения)

Текущая цель — тестовое задание и один инстанс. Для роста нагрузки известные шаги:

1. Фоновые задачи → внешняя очередь (arq/Celery) + отдельный worker.
2. Rate-limit store → Redis (уже поддержан через `REDIS_URL`) для многих инстансов.
3. Кеш метрик, если чтений станет много.
4. Пул соединений БД под конкретный инстанс-профиль.

Эти пункты **намеренно не реализованы** (YAGNI) и зафиксированы как план, чтобы
не усложнять показательный проект.
