# CLAUDE.md — правила для ИИ в этом проекте

Контекст для AI-ассистента при работе с этой кодовой базой.

## Что это

Backend-сервис формы обратной связи (тестовое задание). Стек: Python 3.12,
FastAPI (async), SQLAlchemy 2.0 + Postgres, Alembic. Полный цикл:
валидация → сохранение → AI-анализ → email → ответ.

## Архитектурные правила (соблюдать)

- **Слоистость строго:** `api → services → repositories → models`. Зависимость
  только сверху вниз. Не тащить SQLAlchemy в сервисы, не класть логику в роутеры.
- **Внешние сервисы — через абстракцию (Strategy):** AI и email имеют интерфейс
  и несколько реализаций, выбор — через `.env`. Новый провайдер = новый класс.
- **Надёжность:** внешние вызовы (AI/SMTP) защищены таймаутом и не роняют запрос;
  для AI — circuit breaker + rule-based fallback.
- **Ответ клиенту быстрый:** тяжёлое (AI, email) — в `BackgroundTasks`, не в
  обработчике запроса.
- **Ошибки:** через доменные исключения `app/core/errors.py` и глобальный handler.
  Не «глотать» исключения молча (кроме осознанного fallback с логом).
- **Логи:** структурные (`logger.info("event", **fields)`), без секретов/PII.
- **YAGNI:** не усложнять «на будущее». Масштабирующие шаги зафиксированы в
  `docs/prr.md`, но не реализуются без потребности.

## Процесс

- Перед коммитом: `ruff check app tests` и `pytest` должны быть зелёными.
- Значимое решение — оформить ADR в `docs/adr/`.
- Изменение контракта API — обновить `docs/api.md` (OpenAPI генерируется сам).
- Тесты гоняются на SQLite; Postgres нужен только для прод/CI-миграций.

## Полезные команды

```bash
poetry run uvicorn app.main:app --reload   # запуск dev
poetry run pytest --cov=app                # тесты + покрытие
poetry run ruff check app tests            # линтер
poetry run alembic upgrade head            # миграции
docker compose up --build                  # весь стек
```
